Comment ça se branche (léger)

Nouvelles tables (3 petites)
  • cantine_abonnements — élève, formule (mensuel/prépayé), tarif, statut, solde
  • cantine_repas — date, élève, présent, coût décompté (le « pointage »)
  • cantine_menus — semaine, jour, plats (jsonb)
Réutilise l'existant (rien à refaire)
  • Paiements & recouvrement → factures & relances cantine
  • Fiche élève → champ régime / allergies
  • Espace parent → solde, menu, paiement mobile
  • RBAC & modules → activation par école, accès Gestion/Secrétaire
Accès (cohérent avec la délégation)
  • Comptable / Secrétaire : abonnés, facturation, paiements cantine
  • Personnel cantine (rôle léger) : uniquement le pointage + menu
  • Parent : consultation solde + menu + paiement
Effort & poids
  • Module lazy-load → 0 impact sur les écoles sans cantine
  • ~4 écrans + 3 tables → léger
  • Version « stock/denrées » = optionnelle, plus tard
Piste commerciale
  • Option +30 000 à 50 000 F/an par école (upsell)
